My Interests: Web Accessibility & Inclusive Design

Why Accessibility Matters

Web accessibility is not just a technical requirement—it's a fundamental principle of ethical web development. It ensures that websites and applications are usable by everyone, including people with disabilities. This is a passion of mine, and I believe it should be at the heart of every web project.

As a developer, I'm committed to creating digital experiences that include rather than exclude. The Web Accessibility Initiative (WAI) guidelines provide the framework, but inclusive thinking must guide the implementation.

Understanding WCAG Standards

The Web Content Accessibility Guidelines (WCAG) are developed by the World Wide Web Consortium (W3C) and provide a set of recommendations for making web content more accessible to people with disabilities. These guidelines are organized around four core principles:

The Four Principles: POUR

Perceivable

Information and user interface components must be presentable to users in ways they can perceive. This means providing text alternatives for images, captions for videos, and ensuring sufficient color contrast.

Operable

User interface components and navigation must be operable. Users must be able to navigate using keyboard alone, have sufficient time to read and use content, and avoid seizure-inducing content.

Understandable

Information and the operation of user interfaces must be understandable. Text must be readable and comprehensible, pages must appear and operate in predictable ways, and users must be helped to avoid and correct mistakes.

Robust

Content must be robust enough that it can be interpreted reliably by a wide variety of user agents, including assistive technologies. This means using valid, semantic HTML and following web standards.

WCAG Level Compliance

WCAG standards are organized into three levels of conformance, each building on the previous one:

Level A

Essential

The minimum level of accessibility. Addresses the most critical barriers.

Level AA

Recommended

Removes significant barriers for most people. The target for most web content today.

Level AAA

Advanced

Enhanced accessibility for specialized content. Not always practical for all websites.

Most modern organizations aim for WCAG 2.1 Level AA compliance, which addresses the majority of accessibility concerns while remaining practical to implement.

Practical Accessibility Techniques

1. Semantic HTML

Using semantic HTML elements (<nav>, <main>, <article>, <footer>) helps screen readers understand page structure and meaning.

<!-- Good: Semantic -->
<nav><a href="home">Home</a></nav>
<main>
<article>
<h1>Page Title</h1>
<p>Content here</p>
</article>
</main>

2. Color Contrast

Text must have sufficient contrast against its background. WCAG AA requires a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text.

Testing Tools: WebAIM Contrast Checker, Lighthouse, or axe DevTools can help measure contrast ratios on your website.

3. Image Accessibility

All images should include descriptive alt text that conveys the image's purpose. For decorative images, use empty alt attributes.

<!-- Informative image -->
<img src="chart.png" alt="Sales growth chart showing 25% increase in Q3">

<!-- Decorative image -->
<img src="divider.png" alt="">

4. Keyboard Navigation

All interactive elements must be accessible via keyboard. This includes providing visible focus indicators for links and buttons.

5. Form Accessibility

Forms must have properly associated labels, clear error messages, and instructions for completing required fields.

Real-World Impact

Why Accessibility Benefits Everyone

While accessibility features are essential for people with disabilities, they benefit a much broader audience:

  1. Voice Navigation: Useful for people with mobility impairments, but also helpful while driving or multitasking
  2. Captions: Help deaf users, but also people in noisy environments or using devices without sound
  3. Simple Language: Assists people with cognitive disabilities, but also non-native speakers and mobile users on slow connections
  4. Large Text: Essential for people with low vision, but helpful for aging users and anyone reading on small screens
  5. High Contrast: Aids people with color blindness and improves readability for everyone

Future Learning Goals

My interest in accessibility extends beyond compliance. I'm committed to continuous learning in these areas:

  • WCAG 2.2 and emerging accessibility standards
  • Advanced ARIA (Accessible Rich Internet Applications) patterns for complex interfaces
  • User testing with assistive technology and disabled users

  • Building an inclusive design culture within development teams
  • Automating accessibility testing while understanding its limitations
